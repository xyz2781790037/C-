在模板（ template ）中， class 和 typename 在很多情况下可以互换，但也有些细微差别。
 
 class 是较早期用于模板参数的关键字。比如定义一个简单的模板类， template <class T> ，这里 T 可以是任何类型，像 int 、 double 、 std::string 或者自定义的类类型。
 
 typename 除了能完成和 class 一样的功能外，还有一个额外的用途。当在模板内部访问一个依赖于模板参数的类型时，必须使用 typename 来告诉编译器这是一个类型。例如，有一个模板类包含一个嵌套的类型定义，在模板外部定义成员函数时，需要用 typename 来明确这个嵌套类型是一个类型，而不是别的成员变量。
 
总的来说，只是定义简单的模板参数类型， class 和 typename 基本相同；但涉及到模板内部依赖类型的访问时，就需要 typename 来消除歧义。
